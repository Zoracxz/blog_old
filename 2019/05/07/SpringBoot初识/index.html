SpringBoot 初识

Spring Boot 概述

Spring Boot 是所有基于 Spring 开发的项目的起点。Spring Boot 的设计是为了让你尽可能快的跑起来 Spring 应用程序并且尽可能减少你的配置文件。

Spring Boot 快速搭建

第一步：新建项目

  • 选择 Spring Initializr ，然后选择默认的 url 点击【Next】：

  • 然后修改一下项目的信息：

  • 勾选上 Web 模板：

  • 选择好项目的位置，点击【Finish】：选择好项目的位置，点击【Finish】：

  • 如果是第一次配置 Spring Boot 的话可能需要等待一会儿 IDEA 下载相应的 依赖包，默认创建好的项目结构如下：

DemoApplication： 一个带有 main() 方法的类，用于启动应用程序
DemoApplicationTests：一个空的 Junit 测试文件
application.properties：一个空的 properties 文件，可以根据需要添加配置属性
pom.xml： Maven 构建说明文件

第二步：代码详解

  • 整个项目完成后的结构图：

  1. java 文件夹（代码）：

    • com.zora.example:自定义包名
    • controller：控制器，
    • entity：实体，
    • mapper：与数据库进行操作的类，
    • service：处理后台逻辑，
    • ExampleApplication：应用程序启动类

  2. resources 文件夹（资源）：

    • mybatis：与数据库进行相关操作，
    • mapper：具体数据库操作的xml，
    • mybatis-config.xml：mybatis配置文件，
    • static：图片，css等静态资源文件，
    • templates：放html等模板，
    • application.properties：常量，数据库配置等放在这里。
    • messages.properties：thymeleaf模板引擎的配置文件。
  • Controller

用@Controller注解使spring框架可以识别它，MainController类中有AddPart属性，使用@Autowired自动注入，这个类中有两个方法，一个是addEmp,@PostMapping注解表示用post提交,url为“/addemp”,方法里面是具体的逻辑操作，将emp存入数据库中，然后返回首页。另一个方法index,有一个参数keyword，返回ModelAndView，方法中如果没有keyword就查询全部数据，如果有keyword就查找特定的数据。

  • Entity：

这两个是数据库对应的实体类，每个属性都有对应的get和set方法,由于get和set方法太长，我把它删了。请自行自动生成。

  • Service：

service的接口，这样做的目的是为了解耦。

实现了AddPart接口，完成了具体的逻辑。

  • Mapper：

service中调用这个类与数据库进行操作。

在启动类中添加注解@MapperScan("com.zora.example.mapper")
MapperScan注解会在SpringBoot启动的时候扫描mapper包，并根据xml自动生成对应的实现类

  • Resources：

    1. Mapper

    2. html

      一些页面的设计代码。由于主要讲的是spring boot，有关于页面设计的部分就不放了，想要源码的旁友们，我把代码放在了我的GitHub上。

    3. properties

      application.properties 文件中，server.port=8082改变端口号，取消thymeleaf缓存，数据库的相关配置，以及mapper对应文件夹的配置。

      messages.properties文件中，一些常量的赋值。
